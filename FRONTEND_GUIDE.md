# FactGuardian 前端仪表盘使用指南

## 1. 功能概述
我为您设计并实现了一个基于 React + Tailwind CSS 的现代化仪表盘，完美对接了后端的智能体能力：

- **智能上传区**: 支持拖拽上传，实时显示分析进度。
- **全流程可视化**: 清晰展示 "事实提取 -> 冲突检测 -> 联网校验" 的 AI 思考链路。
- **矛盾冲突雷达 (Conflict Detect)**:
  - 侧边高亮显示前后文矛盾。
  - 区分冲突等级（高/中/低），优先处理严重问题。
- **溯源校验中心 (Source Verify)**:
  - 显示联网搜索到的证据片段。
  - 红色/绿色直观标记事实真伪。
  - **纠错建议**: 当发现错误时，AI 会直接给出正确的数据建议。

## 2. 启动方式 (推荐)

由于已经全面容器化，您只需一条命令即可同时启动前端和后端：

```powershell
docker-compose up --build
```

> **注意**: 首次构建可能需要几分钟下载 Node.js 和 Python 镜像。由于我们在上一步配置了国内镜像源，速度应该是有保障的。

## 3. 访问仪表盘

启动成功后（看到 `factguardian-frontend` 和 `factguardian-backend` 都运行中），请在浏览器访问：

**[http://localhost:3000](http://localhost:3000)**

## 4. 使用流程演示
1. **上传**: 点击上传您的 `test_data_simple.txt` 或其他文档。
2. **等待分析**: 界面会出现动态加载层，显示 Log 实时进度。
3. **查看结果**:
   - **顶栏**: 看到提取了多少事实，发现了多少冲突。
   - **左侧列表**: 逐个审查 "矛盾" 和 "事实谬误"。
   - **AI 分析**: 阅读每一条冲突下的 "AI 分析" 解释，理解为何判定为矛盾。
