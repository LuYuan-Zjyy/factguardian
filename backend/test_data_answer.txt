test_data 实际错误如下
一、事实错误（3 处）
（一）技术认知错误
问题原文：技术及功能适配问题中提及 “小程序采用 Java 技术开发，医院系统采用 Python 技术开发，因适配方案缺失导致接口对接受阻”；优化技术方案中又提到 “Java 与 Python 技术接口可通过中间件实现兼容，无需更换开发技术”。
错误本质：前后对跨语言技术兼容性的认知矛盾，实际 Java 与 Python 接口可通过中间件兼容，属于技术常识错误，且前文将接口受阻归因于技术语言差异，与实际技术逻辑不符。
（二）合规手续事实错误
问题原文：项目进度推进中称 “前期筹备工作已全部完成”；政策及合规性问题中披露 “线下服务站点装修工程未办理施工许可证即开工，施工许可手续并未办结”。
错误本质：前期筹备工作包含施工许可办理，此处表述与实际手续办理情况相悖，属于事实性遗漏错误。
（三）政策依据错误
问题原文：项目建设背景依据《智慧社区建设指南（2023 年版）》推进；政策及合规性问题中提及 “部分建设内容与《智慧社区建设指南（2024 年修订版）》不符”，且未提前跟进政策更新。
错误本质：项目推进未同步跟进最新政策文件，导致建设内容不符合现行标准，属于政策依据滞后的事实错误。
二、表述冲突（8 处）
（一）进度节点冲突（3 处）
竣工时间冲突：
前文：“新出现的分项工程滞后问题，若不能及时解决，可能导致项目整体竣工时间延迟至 2026 年 4 月”；
后文：“计划将项目竣工时间调整为 2026 年 3 月 20 日，较原计划延迟 20 天，仍能确保 3 月底前投入试运行”；
冲突点：两处对整体竣工时间的预判不一致，一处提及可能延迟至 4 月，一处明确调整为 3 月 20 日。
居民协调工作冲突：
前文：“剩余 3 个社区的硬件设备安装工作因社区居民协调问题延迟推进，目前已完成协调工作”；
后文：“有 3 个社区的部分居民反对在楼道安装监控设备，认为侵犯个人隐私，导致设备安装工作延迟”；
冲突点：居民协调工作的完成状态表述矛盾，一处称已完成协调，一处称因居民反对仍延迟。
装修进度表述冲突：
前文：“装修工程已于 8 月中旬启动，目前已完成总装修进度的 70%，预计 10 月底可全部竣工”；
后文：“线下服务站点装修工程进度仅完成 70%，较计划进度滞后 5 个百分点”；
冲突点：前文未提及装修进度滞后，仅说明当前进度及预计竣工时间，后文明确指出进度滞后，表述不一致。
（二）资金相关冲突（3 处）
资金缺口表述冲突：
前文：“目前资金周转正常，无资金缺口问题”；
中后段：“自筹资金仅到位 50%，较计划中期到位比例低 20 个百分点，可能引发项目停工风险”；
后文整改：“此前‘无资金缺口’的表述将调整为‘阶段性资金缺口可控，通过多种方式可解决’”；
冲突点：三处对资金缺口的表述逐步反转，从 “无缺口” 到 “存在停工风险”，再到 “阶段性缺口可控”，逻辑矛盾。
资金管控目标冲突：
前文：“项目初期制定的‘资金使用全程可追溯、无超支风险’的管控目标”；
后文：“存在部分费用超支情况，人员及办公费用超支 23 个百分点，人脸识别门禁设备超支 4.5%，部分报销单据缺乏明细导致资金使用追溯性不足”；
冲突点：初期管控目标与实际资金使用情况相悖，存在超支及追溯性不足问题，与目标冲突。
费用支出表述冲突：
前文：“装修及场地改造费用支出 150 万元，占该部分预算的 50%”；
后文：“线下服务站点装修工程进度仅完成 70%，较计划进度滞后 5 个百分点”；
冲突点：费用支出占比（50%）与装修进度（70%）不匹配，进度与资金投入比例失衡，表述存在逻辑冲突。
（三）功能目标与实际效果冲突（1 处）
医疗预约功能冲突：
前文：便民服务模块目标是 “实现医疗预约闭环服务”，且试运营中包含医疗预约功能；
后文：“医疗预约功能无法与我市三甲医院的挂号系统对接，居民线上预约后仍需线下排队挂号”；
冲突点：功能目标与实际运营效果不符，闭环服务未实现，且试运营成果与后续问题披露矛盾。
（四）前期筹备完整性冲突（1 处）
筹备工作表述冲突：
前文：“前期筹备工作已全部完成，项目立项、可行性研究报告审批、招投标等工作均于 2025 年 2 月底前完成”；
后文：“线下服务站点装修工程未办理施工许可证即开工，施工许可手续并未办结”；
冲突点：施工许可办理属于前期筹备核心环节，前文称筹备工作全部完成，后文披露该手续未办结，表述矛盾。
